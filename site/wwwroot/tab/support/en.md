For app issues, general feedback, and feature requests, use the project issue tracker:

[Open a GitHub issue](https://github.com/deemoe404/onemorecap/issues)

For privacy-sensitive questions, email [me@dee.moe](mailto:me@dee.moe). Do not post subtitle files, movie files, personal data, or private logs in a public issue.

## Before reporting a problem

Please include:

- macOS version.
- One More Cap version.
- Whether you are using the App Store build or the GitHub build.
- Subtitle file type, such as `.srt`, `.vtt`, or `.webvtt`.
- Whether QuickTime Player is open and playing the movie you want to sync.

Do not attach private movie files or private subtitle files unless you intentionally want to share them publicly in the issue.

## QuickTime permission

If Sync cannot read QuickTime Player, open macOS System Settings, go to Privacy & Security, then Automation, and allow One More Cap to read QuickTime Player.

The app remains usable without that permission, but Sync cannot automatically calibrate timing from QuickTime Player.

## Supported subtitle files

One More Cap currently supports SRT and WebVTT subtitle files:

- `.srt`
- `.vtt`
- `.webvtt`

ASS/SSA styling and OCR are not part of the current scope.

## Privacy

Subtitle files stay on your Mac. See the [Privacy Policy](?tab=privacy&lang=en) for details.
